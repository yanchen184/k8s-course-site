## 完整因果鏈（七堂課一條線）

```
手動部署太慢太容易出錯
→ 需要操作伺服器 → Linux 基礎（終端機、檔案、網路）
→ 環境不一致「在我電腦可以跑」→ Docker（容器化）
→ 每次都手動打指令 → Dockerfile（自動打包）
→ 多個容器要一起管 → Docker Compose
→ 一台機器扛不住、容器太多管不動 → Kubernetes
→ 要跑容器 → Pod
→ Pod IP 會變 → Service
→ 地址太醜 → Ingress
→ 設定寫死 → ConfigMap → 密碼明文 → Secret
→ 資料消失 → Volume → PV/PVC
→ 單點故障 → Deployment → 滾動更新 → 回滾
→ DB 有狀態 → StatefulSet
→ 服務卡死沒人知 → Probe
→ 資源被吃光 → Resource limits → HPA 自動擴縮
→ 誰都能刪 → RBAC
→ 全通不安全 → NetworkPolicy
→ 從零部署完整生產系統
```

## 課程簡介

### 開頭 Hook

你想把程式部署到伺服器上。

第一個問題：你不會操作 Linux。所以你學了終端機、檔案管理、網路設定。

第二個問題：你在本機跑得好好的，搬到伺服器就壞了。「在我電腦可以跑啊！」所以你學了 Docker，把環境打包成容器。

第三個問題：容器越來越多，一台機器扛不住了，凌晨三點機器掛了你被電話叫醒。所以你學了 Kubernetes。

**這堂課的每一個工具、每一個概念，都是因為上一步出了問題才學的。**

### 教學方法

**因果鏈教學法** — 市面上的課程大多是「Linux 是什麼」「Docker 是什麼」「K8s 是什麼」的概念介紹式。這堂課不一樣。

這堂課用一條因果鏈貫穿七堂課：

- 你不會操作伺服器 → 所以學 **Linux**
- 環境不一致 → 所以學 **Docker**
- 容器太多管不動 → 所以學 **Kubernetes**
- Pod IP 會變 → 所以需要 **Service**
- 設定寫死要重 build → 所以需要 **ConfigMap**
- 只有一個 Pod 掛了就停 → 所以需要 **Deployment**

你不只知道「是什麼」，你知道「為什麼」。不只會用，你知道為什麼要用。

### 七堂課完整路徑

| 堂次 | 主題 | 你學完能做什麼 |
|:---:|------|--------------|
| 1 | **Linux 基礎** | 會操作伺服器：終端機、檔案、權限、網路 |
| 2 | **Docker 基礎** | 會跑容器：Image、Container、Port、Volume |
| 3 | **Docker 進階** | 會打包和編排：Dockerfile、Docker Compose |
| 4 | **K8s 全貌 + Pod** | 會在 K8s 上跑容器，會讓 K8s 管一群 |
| 5 | **Deployment + Service** | 會讓瀏覽器連到 K8s 裡的服務 |
| 6 | **Ingress + 配置 + 持久化** | 會部署帶資料庫的完整網站 |
| 7 | **生產就緒 + 總複習** | 會從零獨立建一整套生產系統 |

### 適合誰

- 想學容器技術但不知道從哪開始的工程師
- 聽過 Docker / K8s 但「看完教學還是不知道為什麼要學」的人
- 想在履歷上加上 Linux + Docker + Kubernetes 技能的開發者
- 後端工程師想轉 DevOps / SRE

### 不適合誰

- 已經在生產環境操作 K8s 的資深 SRE（這堂課是入門到中階）
- 想學特定雲平台操作（AWS EKS / GCP GKE）的人（這堂課教的是 K8s 本身）

### 學完你會得到

**Linux**
- 能在終端機裡操作檔案、管理權限、設定網路
- 能用 SSH 連到遠端伺服器

**Docker**
- 能用 Dockerfile 打包自己的應用程式
- 能用 Docker Compose 管理多個容器

**Kubernetes**
- 能在 K8s 上部署、擴縮、更新、回滾服務
- 能排除常見的 Pod 錯誤
- 能用 Service + Ingress 讓外部用域名連到服務
- 能用 ConfigMap / Secret / PV / PVC 管理設定、密碼、資料
- 能用 Probe + HPA + RBAC + NetworkPolicy 讓服務生產就緒
- 能從零獨立建一整套 K8s 生產系統
