## 投影片 7：Linux 是什麼？
**預估時間：5 分鐘**

好，讓我們正式進入主題。Linux 到底是什麼？

Linux 是一個開源的作業系統核心，由 Linus Torvalds（林納斯·托瓦茲）在 1991 年創建。當時他只是一個 21 歲的芬蘭赫爾辛基大學學生。

要理解 Linux 的誕生，我們要先知道一個更老的作業系統叫做 Unix。Unix 是 1969 年由貝爾實驗室開發的，可以說是現代作業系統的老祖宗。Unix 的設計哲學非常優雅：「做一件事，把它做好」、「一切都是檔案」、「用小工具組合成大功能」。這些設計理念到今天還在影響我們。

但 Unix 有個大問題：它是商業軟體，授權費非常貴，一套要幾萬美金，一般人和學生根本負擔不起。

那 1991 年的時候，市面上有哪些作業系統呢？有商用的 Unix、微軟的 MS-DOS、剛推出的 Windows 3.0，還有蘋果的 macOS。對了，那時候的 Windows 3.0 嚴格來說還不算真正的作業系統，它只是跑在 DOS 上面的圖形介面程式。真正獨立的 Windows 要等到 1995 年的 Windows 95 才出現。

Linus 當時用的是一套叫做 MINIX 的教學用作業系統，是荷蘭教授 Andrew Tanenbaum 為了教學目的開發的。MINIX 程式碼開放，但授權有限制，不能自由修改和散布。Linus 覺得 MINIX 功能不夠用，又買不起商用的 Unix，乾脆自己從零寫一個！

他把原始碼公開到網路上分享，沒想到引來全世界的開發者一起貢獻。30 多年後的今天，Linux 已經成為全球最重要的作業系統之一。

Linux 這個名字是 Linus 加上 Unix 的組合。雖然 Linux 借鑒了 Unix 的設計哲學和操作方式，但程式碼是 Linus 從零開始寫的，沒有複製任何一行 Unix 的程式碼。這點很重要，因為這樣 Linux 才能用開源授權自由散布，不會有法律問題。

最後要特別說明一下，嚴格來說，Linux 只是作業系統的「核心」，也就是 kernel。我們平常說的 Linux 作業系統，其實是 Linux kernel 加上其他工具程式組合而成的。

## 投影片 8：Linux vs Windows
**預估時間：6 分鐘**

現在讓我們來比較 Linux 和 Windows 的差異。

第一個差異是授權方式。Linux 是開源免費的，任何人都可以下載、使用、修改。Windows 是商業軟體，需要購買授權。這也是為什麼大部分伺服器都選擇 Linux，因為可以省下大量的授權費用。

第二個差異是操作介面。Windows 主要靠滑鼠點選圖形介面，而 Linux 傳統上是用命令列操作。雖然現在 Linux 也有圖形介面，但在伺服器環境，我們幾乎都是用命令列。

第三個差異是檔案系統。Windows 用的是 NTFS，Linux 常用的有 ext4、xfs 等。

第四個差異是路徑的寫法。Windows 用反斜線 backslash 來分隔資料夾，反斜線就是鍵盤上 Enter 鍵上面那個，長這樣 `\`。而且 Windows 通常有 C 槽、D 槽這種概念。Linux 用正斜線 slash，就是問號那個鍵，長這樣 `/`，而且整個系統只有一個根目錄，用斜線開頭。

你可能會問，為什麼這兩個系統不統一用同一種斜線？這是歷史原因造成的。Unix 在 1969 年就先用了正斜線 `/`。後來 1981 年 IBM PC 的 DOS 系統也想用正斜線，但當時已經把 `/` 用在命令列參數了，例如 `dir /w`，所以只好改用反斜線 `\`。Windows 繼承了 DOS 的習慣，就一直用到現在。現在兩邊都有幾十年的軟體和習慣，誰都改不了了。

這個差異會造成跨平台問題。如果你在程式碼裡寫死了路徑，例如 `C:\Users\data`，這段程式移到 Linux 就會出錯。這就是為什麼在程式開發中，要用程式語言提供的路徑處理工具，像是 Python 的 `os.path.join`、Node.js 的 `path.join`，而不是直接用字串拼接路徑。在 Kubernetes 的 YAML 設定中，路徑一律用 Linux 格式，也就是正斜線。

最後一個很重要的差異是大小寫，這是從 Windows 過來的新手幾乎必踩的雷。在 Windows 上，你建立了 `Config.txt`，用 `type config.txt` 也能讀取到，因為 Windows 不區分大小寫。但在 Linux 上，`Config.txt`、`config.txt`、`CONFIG.TXT` 是三個完全不同的檔案。

舉個實際的例子，如果你建立了 `Dockerfile`（D 大寫），執行 `docker build` 的時候它預設找 `Dockerfile`，剛好沒問題。但如果你命名成 `dockerfile`（全小寫），就找不到了。在 Kubernetes 的 YAML 設定檔中，key 的大小寫也完全固定，一個字母寫錯大小寫，整個設定就無效。

那檔名可以有空白嗎？技術上可以，但強烈建議不要。在 Linux 上，如果你建立了一個叫 `my file.txt` 的檔案，要讀取它的時候就麻煩了。因為命令列是用空白來分隔參數的，如果你輸入 `cat my file.txt`，系統會以為你要讀取兩個檔案：一個叫 `my`，一個叫 `file.txt`。

要正確讀取有空白的檔名，你必須用引號包起來：`cat "my file.txt"`，或者用反斜線跳脫：`cat my\ file.txt`。每次操作都要多打這些符號，非常麻煩。而且在寫腳本的時候，有空白的檔名更容易出問題，很多腳本沒有正確處理空白，一跑就爆掉。

所以從今天起，養成好習慣：在 Linux 上命名檔案和目錄，全部用小寫，用連字號或底線分隔，例如 `my-config.yaml` 或 `deploy_script.sh`，不要用空白。

## 投影片 9：為什麼伺服器用命令列？
**預估時間：4 分鐘**

為什麼伺服器環境都用命令列？四個原因：

第一，資源。圖形介面需要顯示卡和記憶體來渲染視窗、動畫。伺服器的資源應該用在處理請求，不是畫面。

第二，效率。遠端桌面要傳輸整個畫面，延遲高。命令列只傳文字，幾乎沒有延遲。

第三，自動化。舉個例子：假設你管理 50 台 Nginx，要把 `worker_connections` 從 1024 改成 4096。

GUI：一台一台連，50 台要 2-3 小時，無法確保設定一致。

命令列：

```bash
for server in $(cat servers.txt); do
  ssh user@$server "sed -i 's/1024/4096/' /etc/nginx/nginx.conf && systemctl restart nginx"
done
```

2-3 分鐘，50 台完成，有 log 追溯。

第四，可複製性。指令可以記錄、版本控制，其他人可以重現你的操作。

## 投影片 12：Linux 檔案系統結構
**預估時間：4 分鐘**

休息回來，讓我們繼續。

Linux 有一個很重要的設計哲學：一切都是檔案。不只是文件是檔案，連硬碟、鍵盤、網路介面，在 Linux 看來都是檔案。

Linux 的檔案系統是一個樹狀結構，最頂端是根目錄，用一個斜線表示。

讓我介紹幾個重要的目錄：

bin 目錄放的是基本指令，像是 ls、cp、mv 這些。

etc 目錄放的是系統設定檔，以後我們改設定都會在這裡操作。

home 目錄是一般使用者的家目錄。每個使用者在這裡有自己的資料夾。比如你的帳號是 student，那你的家目錄就是 /home/student。

var 目錄放的是可變資料，最常用到的是 /var/log，系統日誌都在這裡。

tmp 是暫存目錄，重開機會被清空。

root 目錄是系統管理員 root 使用者的家目錄，注意它不在 /home 下面。

## 投影片 13：使用者與權限
**預估時間：4 分鐘**

Linux 是一個多使用者系統，不同的人登入有不同的權限。

最重要的使用者是 root，這是系統管理員帳號，擁有最高權限，可以做任何事情。包括刪除整個系統！所以我們有一句話：權力越大，責任越大。

日常操作我們會使用一般使用者帳號，只有在需要的時候才切換到 root 權限。

每個檔案都有權限設定，決定誰可以讀取、寫入、執行。

這串字母看起來可能很複雜，讓我解釋一下。第一個字元表示檔案類型，減號表示一般檔案，d 表示目錄。

接下來每三個字元一組。rwx 代表讀取 read、寫入 write、執行 execute。第一組是檔案擁有者的權限，第二組是所屬群組的權限，第三組是其他人的權限。

減號表示沒有那個權限。比如 r-- 就是只能讀取，不能寫入也不能執行。

## 投影片 16：你的第一個 Linux 指令
**預估時間：5 分鐘**

大家都登入成功了嗎？太好了！

現在讓我們來執行第一個 Linux 指令。

第一個指令是 whoami，就是「我是誰」的意思。輸入 whoami 按 Enter，它會告訴你現在登入的使用者名稱。

第二個指令是 pwd，全名是 print working directory，顯示你現在在哪個目錄。應該會顯示 /home/student，這是你的家目錄。

第三個指令是 ls，這是最常用的指令之一，用來列出目錄的內容。就像 Windows 的「開啟資料夾」看到裡面有什麼檔案。

大家現在就在自己的終端機試試看這三個指令。

**預估時間：2 分鐘**

ls 列出目錄內容，cd 切換目錄，pwd 顯示目前位置，mkdir 建立新目錄，cp 複製檔案，mv 移動或重新命名檔案，rm 刪除檔案，cat 顯示檔案內容。
