## 投影片 3：目錄切換 - cd
**預估時間：5 分鐘**

cd 是 change directory，切換目錄用的。

進入某個目錄：

```bash
cd Documents
```

進去之後用 pwd 確認位置。

回到上一層目錄，用兩個點：

```bash
cd ..
```

不管你現在在哪裡，直接回到家目錄：

```bash
cd ~
```

或是直接打 cd 不加任何參數：

```bash
cd
```

回到上一個待過的目錄（很實用！）：

```bash
cd -
```

這個減號的功能就像瀏覽器的「上一頁」，在兩個目錄之間切換超方便。

大家現在練習：進入某個目錄，再用 `cd -` 切回來。

## 投影片 4：相對路徑 vs 絕對路徑
**預估時間：5 分鐘**

這是很重要的觀念，後面寫 Dockerfile 和 Kubernetes YAML 都會用到。

**絕對路徑**：開頭是斜線 `/`，從根目錄開始算。不管你現在在哪裡，絕對路徑永遠指向同一個位置。

```bash
cd /home/student/Documents
```

**相對路徑**：相對於你目前位置的路徑。

```bash
cd Documents
```

這個 Documents 是相對於你現在的位置。如果你在 `/home/student`，Documents 就是 `/home/student/Documents`。如果你在別的地方，Documents 就是那裡的 Documents。

特殊符號：
- `.` 代表目前目錄
- `..` 代表上一層目錄
- `~` 代表家目錄

舉個例子，如果你現在在 `/home/student/a/b/c`，想去 `/home/student/a/d`：

絕對路徑：`cd /home/student/a/d`
相對路徑：`cd ../../d`（往上兩層，再進入 d）

**實際應用**：在 Dockerfile 裡面，你會看到這樣的指令：

```dockerfile
COPY ./src /app/src
```

這裡的 `./src` 就是相對路徑，相對於 Dockerfile 所在的目錄。`/app/src` 是絕對路徑，是容器內的目錄。

## 投影片 5：建立目錄 - mkdir
**預估時間：4 分鐘**

mkdir 是 make directory 的縮寫。

```bash
mkdir myfolder
```

這樣就建立了一個叫 myfolder 的目錄。用 ls 確認一下。

如果要一次建立多層目錄，例如 `a/b/c`，要加 `-p` 參數：

```bash
mkdir -p a/b/c
```

沒有 `-p` 的話，如果 a 不存在，指令會失敗。`-p` 的意思是 parent，會自動建立需要的父目錄。

`-p` 還有另一個好處：如果目錄已經存在，不會報錯。

```bash
mkdir myfolder      # 第二次執行會報錯：目錄已存在
mkdir -p myfolder   # 不會報錯，靜默成功
```

這在寫腳本的時候很有用，不用先檢查目錄存不存在。

一次建立多個目錄：

```bash
mkdir dir1 dir2 dir3
```

配合大括號展開（brace expansion）：

```bash
mkdir -p project/{src,test,docs}
```

這會建立 project/src、project/test、project/docs 三個目錄。

## 投影片 6：建立檔案 - touch 與其他方法
**預估時間：4 分鐘**

touch 可以建立空白檔案：

```bash
touch myfile.txt
```

如果檔案已經存在，touch 不會覆蓋內容，只會更新它的修改時間。

這個指令名稱很有趣，touch 是「觸碰」的意思。想像你輕輕碰了一下檔案，檔案的時間戳記就更新了。

**其他建立檔案的方法**：

用 echo 寫入內容並建立檔案：

```bash
echo "Hello World" > hello.txt
```

`>` 是重導向，把 echo 的輸出寫入檔案。如果檔案存在會覆蓋。

用 `>>` 是追加，不會覆蓋：

```bash
echo "Line 2" >> hello.txt
```

建立空檔案的另一種方式：

```bash
> empty.txt
```

這個技巧在清空檔案內容時很有用：

```bash
> /var/log/myapp.log   # 清空 log 檔案但保留檔案
```

## 投影片 7：複製 - cp
**預估時間：4 分鐘**

cp 是 copy 的縮寫。

```bash
cp myfile.txt myfile_backup.txt
```

這樣就複製了一份。

如果要複製到另一個目錄：

```bash
cp myfile.txt practice/
```

如果要複製整個目錄，要加 `-r` 參數（recursive，遞迴）：

```bash
cp -r myfolder myfolder_backup
```

沒有 `-r` 的話，複製目錄會失敗。因為目錄裡面可能還有子目錄和檔案，需要遞迴處理。

保留檔案屬性（時間、權限）用 `-p`：

```bash
cp -p important.conf important.conf.bak
```

常用組合 `-rp`，複製目錄並保留所有屬性：

```bash
cp -rp /etc/nginx /backup/nginx
```

互動模式 `-i`，覆蓋前會詢問：

```bash
cp -i source.txt dest.txt
```

如果 dest.txt 已存在，會問你要不要覆蓋。

## 投影片 8：移動與重新命名 - mv
**預估時間：4 分鐘**

mv 是 move 的縮寫，有兩個用途。

**移動檔案**：

```bash
mv myfile.txt practice/
```

這樣 myfile.txt 就移到 practice 目錄裡面了。

**重新命名**：

```bash
mv oldname.txt newname.txt
```

mv 的判斷邏輯：第二個參數如果是已存在的目錄，就是移動。如果不是目錄，就是重新命名。

Linux 沒有專門的 rename 指令，重新命名就是用 mv。

**批次重新命名的技巧**：

假設你要把所有 `.txt` 改成 `.md`：

```bash
for f in *.txt; do mv "$f" "${f%.txt}.md"; done
```

這個今天不細講，但讓大家知道有這個可能性。

同樣有 `-i` 互動模式：

```bash
mv -i source.txt dest.txt
```

## 投影片 9：刪除 - rm（重要！）
**預估時間：6 分鐘**

rm 是 remove 的縮寫。這是最危險的指令，請特別注意。

刪除檔案：

```bash
rm myfile.txt
```

刪除目錄要加 `-r`：

```bash
rm -r myfolder
```

有時候系統會問你確認，如果要跳過確認直接刪：

```bash
rm -rf myfolder
```

`-f` 是 force 強制的意思。

**重點：Linux 沒有資源回收桶**，rm 刪掉就真的刪掉了，沒辦法復原。不像 Windows 可以去資源回收桶撈回來。

講到 rm -rf，有一個經典的「毀滅指令」你們一定要知道：

```bash
rm -rf /*
```

這個指令的意思是：強制刪除根目錄下的所有東西。

- `/` 是根目錄
- `*` 是萬用字元，代表所有檔案
- `-rf` 是強制遞迴刪除

執行這個指令，整個系統就沒了。作業系統、所有程式、所有資料，全部清空。電腦會在你面前慢慢死去，正在執行的程式一個一個消失，最後連 rm 自己都被刪掉。

這個指令在網路上很有名，有些人會惡作劇叫新手執行這個「清理系統」或「加速電腦」的神奇指令。所以如果有人叫你在 Linux 上執行什麼看不懂的指令，先 Google 一下那個指令是幹嘛的。

現代的 Linux 發行版有做保護，執行這個指令會跳警告或直接拒絕。但如果你加了 `--no-preserve-root` 參數，或是用其他方式繞過，還是會執行。

**安全刪除的習慣**：

1. 刪除前先用 `ls` 確認
2. 重要資料先 `cp` 備份
3. 養成用 `-i` 的習慣：`rm -ri folder/`

## 投影片 10：萬用字元 - Wildcards
**預估時間：5 分鐘**

萬用字元讓你可以一次匹配多個檔案，非常實用。

**星號 `*`**：匹配任意數量的任意字元

```bash
ls *.txt           # 所有 .txt 檔案
ls log*            # 所有 log 開頭的檔案
ls *2024*          # 檔名包含 2024 的檔案
rm *.log           # 刪除所有 .log 檔案
```

**問號 `?`**：匹配單一個任意字元

```bash
ls file?.txt       # file1.txt, file2.txt, fileA.txt...
ls ???.txt         # 檔名剛好三個字的 .txt 檔案
```

**方括號 `[]`**：匹配括號內的任一字元

```bash
ls file[123].txt   # file1.txt, file2.txt, file3.txt
ls file[a-z].txt   # filea.txt 到 filez.txt
ls file[0-9].txt   # file0.txt 到 file9.txt
```

**大括號 `{}`**：展開列表

```bash
cp config.{txt,bak}        # 等於 cp config.txt config.bak
mkdir {dev,test,prod}      # 建立三個目錄
touch file{1..5}.txt       # file1.txt 到 file5.txt
```

**實際應用範例**：

備份所有設定檔：

```bash
cp /etc/nginx/*.conf /backup/
```

刪除所有暫存檔：

```bash
rm -f /tmp/*.tmp
```

## 投影片 11：檢視檔案內容 - cat
**預估時間：4 分鐘**

cat 是 concatenate 的縮寫，原本是用來連接多個檔案的，但最常用來顯示檔案內容：

```bash
cat myfile.txt
```

如果檔案很長，內容會一次全部倒出來，前面的內容會被洗掉。

**顯示行號**：

```bash
cat -n myfile.txt
```

每一行前面會加上行號，debug 的時候很有用。

**合併多個檔案**（這才是 cat 的本意）：

```bash
cat file1.txt file2.txt > combined.txt
```

**反向顯示**（從最後一行開始）：

```bash
tac myfile.txt
```

注意是 tac，cat 反過來寫。這不是開玩笑，真的有這個指令。

## 投影片 12：檢視檔案 - head 與 tail
**預估時間：5 分鐘**

head 顯示檔案開頭，預設前 10 行：

```bash
head myfile.txt
```

指定行數用 `-n`：

```bash
head -n 20 myfile.txt
```

簡寫：

```bash
head -20 myfile.txt
```

tail 顯示檔案結尾：

```bash
tail myfile.txt
tail -n 20 myfile.txt
```

**tail -f：即時追蹤**

```bash
tail -f /var/log/syslog
```

這個指令會卡住，持續顯示新增的內容。按 `Ctrl + C` 結束追蹤。

這在看 log 的時候超好用。伺服器出問題，你開一個 `tail -f` 盯著 log，一有新錯誤馬上看到。

**追蹤多個檔案**：

```bash
tail -f /var/log/syslog /var/log/auth.log
```

**配合 grep 過濾**：

```bash
tail -f /var/log/syslog | grep ERROR
```

只顯示包含 ERROR 的行。

## 投影片 13：分頁檢視 - less
**預估時間：4 分鐘**

cat 的問題是檔案太長會洗掉前面的內容。less 可以分頁檢視：

```bash
less myfile.txt
```

操作方式：
- `空白鍵` 或 `f`：下一頁
- `b`：上一頁
- `g`：跳到開頭
- `G`：跳到結尾
- `/keyword`：搜尋 keyword，按 `n` 找下一個，`N` 找上一個
- `q`：離開

less 比 more 功能更強，名字的由來是「less is more」（少即是多）。

**實用技巧**：

直接跳到某一行：

```bash
less +100 myfile.txt    # 從第 100 行開始顯示
```

搜尋並跳到第一個結果：

```bash
less +/ERROR logfile    # 開啟並跳到第一個 ERROR
```

## 投影片 14：搜尋檔案 - find
**預估時間：6 分鐘**

find 用來搜尋檔案，功能非常強大。

**依檔名搜尋**：

```bash
find /home -name "*.txt"
```

在 /home 目錄下搜尋所有 .txt 檔案。

**不區分大小寫**：

```bash
find /home -iname "readme*"
```

會找到 README、readme、Readme 等。

**依類型搜尋**：

```bash
find /var -type f    # 只找檔案
find /var -type d    # 只找目錄
```

**依大小搜尋**：

```bash
find /home -size +100M    # 大於 100MB 的檔案
find /tmp -size -1k       # 小於 1KB 的檔案
```

**依修改時間搜尋**：

```bash
find /var/log -mtime -7     # 7 天內修改過的
find /tmp -mtime +30        # 30 天前修改的（可能可以清掉）
```

**找到後執行動作**：

```bash
find /tmp -name "*.tmp" -delete    # 刪除找到的檔案
find /home -name "*.log" -exec rm {} \;   # 用 exec 執行指令
```

**實際案例**：找出佔用空間的大檔案

```bash
find /home -type f -size +100M -exec ls -lh {} \;
```

## 投影片 15：搜尋檔案內容 - grep
**預估時間：6 分鐘**

grep 用來搜尋檔案內的文字內容。

```bash
grep "error" logfile.txt
```

找出 logfile.txt 裡面包含 error 的行。

**不區分大小寫**：

```bash
grep -i "error" logfile.txt
```

會找到 error、Error、ERROR 等。

**顯示行號**：

```bash
grep -n "error" logfile.txt
```

**顯示前後文**：

```bash
grep -A 3 "error" logfile.txt   # 顯示匹配行及後面 3 行
grep -B 3 "error" logfile.txt   # 顯示匹配行及前面 3 行
grep -C 3 "error" logfile.txt   # 前後各 3 行
```

**反向搜尋（排除）**：

```bash
grep -v "debug" logfile.txt    # 顯示不包含 debug 的行
```

**遞迴搜尋整個目錄**：

```bash
grep -r "TODO" /home/project/
```

在整個專案裡搜尋 TODO。

**只顯示檔名**：

```bash
grep -l "error" *.log
```

**配合 pipe 使用**：

```bash
ps aux | grep nginx           # 找 nginx 相關的程序
cat /etc/passwd | grep student   # 找 student 使用者
```

**實際案例**：在 log 裡找錯誤訊息的上下文

```bash
grep -C 5 "Exception" /var/log/app.log
```

## 投影片 19：vim - 模式概念
**預估時間：6 分鐘**

vim 最特別的地方是它有「模式」的概念。

```bash
vim myfile.txt
```

vim 有幾個主要模式：

1. **Normal 模式**（預設）：用來移動游標、複製貼上、刪除、下指令
2. **Insert 模式**：用來輸入文字
3. **Visual 模式**：用來選取文字
4. **Command 模式**：用來執行命令（存檔、離開等）

剛進入 vim 是 Normal 模式。這時候你打字不會輸入文字，而是在下指令。

按 `i` 進入 Insert 模式，左下角會顯示 `-- INSERT --`。這時候才能正常打字。

在 Insert 模式按 `Esc` 回到 Normal 模式。

這是新手最常卡住的地方：進了 vim 一直按鍵盤，怎麼畫面亂跳？因為你在 Normal 模式，按的都是指令，不是在輸入文字。

**進入 Insert 模式的幾種方式**：
- `i`：在游標前插入
- `a`：在游標後插入
- `o`：在下一行插入
- `O`：在上一行插入

記住：**按 i 開始打字，按 Esc 停止打字**。

## 投影片 21：vim - 存檔與離開
**預估時間：4 分鐘**

在 Normal 模式下輸入冒號 `:` 進入 Command 模式，左下角會出現冒號等你輸入：

- `:w` - 存檔（write）
- `:q` - 離開（quit）

**另一個存檔離開的方式**：在 Normal 模式按 `ZZ`（大寫）。

## 投影片 23：權限基礎回顧
**預估時間：3 分鐘**

上午提過，Linux 每個檔案都有權限設定：

```
-rw-r--r-- 1 student student 1234 Jan 1 12:00 myfile.txt
```

前 10 個字元的意思：
- 第 1 個：檔案類型（`-` 檔案、`d` 目錄、`l` 連結）
- 第 2-4 個：擁有者（owner）權限
- 第 5-7 個：群組（group）權限
- 第 8-10 個：其他人（others）權限

r = read（讀取）= 4
w = write（寫入）= 2
x = execute（執行）= 1

這些數字是可以加起來的：
- rwx = 4+2+1 = 7
- rw- = 4+2+0 = 6
- r-x = 4+0+1 = 5
- r-- = 4+0+0 = 4

## 投影片 24：修改權限 - chmod
**預估時間：5 分鐘**

chmod 是 change mode 的縮寫，用來修改檔案權限。

**數字模式**：

```bash
chmod 755 script.sh
```

755 的意思：
- 擁有者：7 (rwx) = 可讀可寫可執行
- 群組：5 (r-x) = 可讀可執行
- 其他人：5 (r-x) = 可讀可執行

常用的權限數字：
- `644`：一般檔案（擁有者可讀寫，其他人只能讀）
- `755`：腳本/執行檔（擁有者全權限，其他人可讀可執行）
- `600`：機密檔案（只有擁有者可讀寫）
- `700`：機密目錄

**符號模式**：

```bash
chmod u+x script.sh    # 給擁有者加上執行權限
chmod g-w file.txt     # 移除群組的寫入權限
chmod o=r file.txt     # 設定其他人只能讀取
chmod a+r file.txt     # 所有人都加上讀取權限
```

- u = user（擁有者）
- g = group（群組）
- o = others（其他人）
- a = all（所有人）

**遞迴修改**：

```bash
chmod -R 755 /var/www/html
```

修改目錄及其下所有檔案的權限。

## 投影片 25：修改擁有者 - chown
**預估時間：4 分鐘**

chown 是 change owner 的縮寫。

```bash
chown student file.txt
```

把 file.txt 的擁有者改成 student。

同時修改擁有者和群組：

```bash
chown student:developers file.txt
```

只修改群組：

```bash
chown :developers file.txt
```

或用 chgrp：

```bash
chgrp developers file.txt
```

遞迴修改：

```bash
chown -R www-data:www-data /var/www/html
```

這在設定 web server 的時候很常用，讓 nginx 或 apache 的使用者可以存取網站目錄。

**注意**：chown 通常需要 root 權限才能執行。

```bash
sudo chown student file.txt
```

## 投影片 28：指令的學習方法
**預估時間：3 分鐘**

Linux 指令非常多，不可能全部背起來。鳥哥的 Linux 私房菜那麼厚一本，你不可能全部記住。

正確的學習方法：需要的時候再查。常用的指令，用久了自然會記住。不常用的，每次查就好。

查指令有幾個方法：

**1. man 指令**

```bash
man ls
```

會顯示 ls 的完整說明書。按 `q` 離開。

**2. --help 參數**

```bash
ls --help
```

比 man 精簡，適合快速查參數。

**3. tldr（推薦安裝）**

```bash
tldr ls
```

tldr 是「Too Long; Didn't Read」的縮寫，給你最常用的範例。比 man 更實用。

**4. Google 或 AI**

「linux how to find files by name」這種問題，Google 或 ChatGPT 回答得很好。

不要覺得查資料很丟臉，工程師每天都在查。重要的是知道怎麼查、知道關鍵字怎麼下。

## 投影片 29：為什麼要學這些指令
**預估時間：4 分鐘**

可能有人會想：這些事情用滑鼠點一點就做完了，為什麼要學指令？

三個你一定會遇到的情境：

**1. 遠端伺服器**

正式環境的伺服器沒有圖形介面，只能用 SSH 連進去下指令。你的公司不會讓你在伺服器上裝圖形介面，那是浪費資源。

**2. Docker 容器**

Docker 容器裡面是精簡的 Linux，連桌面都沒有。不管是寫 Dockerfile 定義環境，還是 `docker exec` 進入容器內部除錯，都需要用指令。

這門課後面會學 Docker，到時候你會發現今天學的指令一直在用。

**3. CI/CD 自動化**

持續整合、持續部署的流程是寫腳本執行的。那些 YAML 檔案裡面的 script 區塊，寫的就是這些指令的組合。

```yaml
# GitHub Actions 範例
- run: |
    mkdir -p dist
    cp -r src/* dist/
    rm -rf dist/test
    chmod 755 dist/deploy.sh
```

看，都是今天學的指令。
